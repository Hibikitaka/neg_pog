\chapter{FSPに対するMBOの解の数の検討}
%%%%%%%%%%%%%%%%%%%

MBOの探索性能は解の数に大きく依存する．解の数が多い程，解の探索範囲が広くなるため良好な解を算出しやすい．
しかし，さらに解の数を増やすと十分な解の探索が行われないため，パフォーマンスが下がる．
言い換えると，解の数に対するMBOのパフォーマンスはU字型の挙動を示す．
そこで，各アルゴリズム一回試行による計算打ち切り時間30nm[ms]に対する適切な解の数を検証する．

また並び替え方法を変更することで解の収束にどのような影響がでるか検証するため，複数の並び替えのもと実験を行った．
並び替えは以下の通りである．

\begin{itemize}
	\item \textgt{MBO(m=10)}
	\\従来通りのMBO
	\item \textgt{MBO(m=1)}
	\\並び替え方は従来通りだが各解の近傍を１回ずつ生成した後並び替え行う．
	\\以下の並び替えも各解の近傍を１回ずつ生成した後別手法の並び替え行う．
	\item \textgt{MBO$_{BELE}$}
	\\並び替え時に先頭の解と解の中の最良の解の場所を入れ替える．
	\item \textgt{MBO$_{Down}$}
	\\解を解の評価値の降順に並び変える．
	\item \textgt{MBO$_{Up}$}
	\\解を解の評価値の昇順に並び替える．
	\item \textgt{MBO$_{Shuffle}$}
	\\解をランダムに並び替える．
\end{itemize}
